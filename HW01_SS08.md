2. Prompt tối ưu
Vai trò

Bạn là một Senior Java Developer có hơn 15 năm kinh nghiệm phát triển hệ thống Core Banking và Spring Boot.

Bối cảnh

SecureBank cần nhập dữ liệu giao dịch từ hệ thống Core Banking cũ. Dữ liệu được cung cấp dưới dạng một chuỗi JSON thô (Raw JSON String). Trong dữ liệu có thể tồn tại nhiều bản ghi lỗi hoặc dữ liệu không hợp lệ.

Nhiệm vụ

Hãy sinh mã Java 17 cho class TransactionParser.

Input

Một chuỗi JSON biểu diễn danh sách giao dịch.

Mỗi phần tử có các trường:

id
amount
status
transactionDate

Ví dụ dữ liệu có thể chứa:

id rỗng
amount là chuỗi "abc"
amount âm
amount bằng 0
status = PENDING
status = CANCEL
transactionDate sai định dạng
transactionDate bị thiếu
Output

Trả về:

List<TransactionDTO>

Trong đó

public record TransactionDTO(
    String id,
    BigDecimal amount,
    String status,
    LocalDateTime localDateTime
){}
Ràng buộc kỹ thuật
Java 17
Spring Boot
Jackson ObjectMapper
Sử dụng ObjectMapper để parse JSON
Không dùng Gson
Không dùng org.json

Kiểm tra dữ liệu:

id không được null hoặc rỗng
amount phải lớn hơn 0
status chỉ nhận SUCCESS hoặc FAILED
transactionDate phải parse được sang LocalDateTime

Nếu bản ghi không hợp lệ:

bỏ qua bản ghi đó
ghi log lỗi
tiếp tục xử lý các bản ghi còn lại
tuyệt đối không làm dừng toàn bộ quá trình import

Yêu cầu xử lý exception đầy đủ.

Trình bày trước khi sinh mã

Không tiết lộ chuỗi suy luận nội bộ.

Thay vào đó:

Tóm tắt cách tiếp cận.
Liệt kê các bước của thuật toán (pseudocode hoặc kế hoạch xử lý).
Giải thích ngắn gọn cách kiểm tra dữ liệu.

Sau đó mới sinh mã Java hoàn chỉnh.

Mã nguồn cần sinh

Sinh đầy đủ:

TransactionDTO (record)
TransactionParser
Các import cần thiết
Logger
Phương thức parseTransactions(String json)
Comment ngắn gọn cho từng bước xử lý
Mã có thể sử dụng trực tiếp trong Spring Boot.
